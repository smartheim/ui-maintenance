Advanced: Get to know rules and timers
  1. Turn on a light by a button press, but only if it is after 6 pm.
    Subject: Rules
    Objectives:
    • Create a rule that changes an item state.
    • Add a condition to it.
    • Let it be triggered by an item state.
  2. Setup a wakeup alarm that turns on the coffee machine and 10 minutes later all lights
    Subject: Timers
    Objectives:
    • Create a one-shot or periodic timer.
    • Create a rule and let it be triggered by a timer.
  3. Turn a light automatically off 5 minutes later after it has been turned on. A motion sensor resets the timer.
    Subject: Dynamic timers
    Objectives:
    • Create a rule that changes an item state and is triggered by an item state.
    • Create a dynamic timer within a rule with an ID based on the item that triggered the rule.
    • Reset a timer by ID within a rule.
Rules and timers are essential for advanced functionality

More and more bindings support Actions that can be run by a rule. Expect even more to come.

You still have scratched only the surface. If you want to integrate a new kind of device via generic HTTP, MQTT, CoAP bindings, you also want to read about Transformations.

We have only used the graphical rule editor so far, to connect triggers with actions on optional conditions. openHAB allows freely written scripts (they are also called NRE Rules in the community). A script can interact with your system, an external service, your hardware or just perform item state changes and react to triggers on conditions. The freedom due to scripts allow you to master any complex scenarios with openHAB.
